Project Overview

VibeMatch is a music discovery platform that uses Natural Language Processing (NLP) and unsupervised machine learning to analyze song lyrics and group them by semantic similarity. Instead of traditional genre-based categorization, VibeMatch creates "vibes" - clusters of songs that share similar themes, emotions, and lyrical content.

The application demonstrates an end-to-end machine learning pipeline: from data ingestion and preprocessing, through transformer-based text encoding, to clustering and interactive visualization - all wrapped in a user-friendly web interface with real-time progress tracking.

Machine Learning Pipeline

The core of VibeMatch is a 4-step ML pipeline that transforms raw song lyrics into meaningful clusters:

Step Process Techniques Used Skills Demonstrated
1 Data Loading & Cleaning Pandas DataFrame operations, text filtering, sampling Data Engineering, ETL
2 BERT Encoding Sentence-BERT, transformer embeddings (384-768 dims) Deep Learning, NLP
3 KMeans Clustering Unsupervised clustering, silhouette analysis ML Algorithms, Model Evaluation
4 Visualization WordCloud generation, stopword removal Data Visualization, NLP

Recommendation Engine

Beyond clustering, VibeMatch includes a cosine similarity-based recommendation engine that finds semantically similar songs based on user input (either raw lyrics or a selected song). This demonstrates information retrieval techniques using vector embeddings.

Challenges & Solutions

Processing Large Text Datasets

Encoding thousands of song lyrics through BERT models is computationally intensive and slow.

Solution

Implemented batch processing with progress callbacks and support for multiple model sizes to balance speed vs. quality.

Real-time User Feedback

Long-running ML pipelines leave users uncertain about progress and completion status.

Solution

Implemented Server-Sent Events (SSE) for real-time streaming updates with automatic polling fallback.

Meaningful Cluster Representation

Cluster numbers alone don't convey the "vibe" or theme of grouped songs.

Solution

Created custom stopword filtering and WordCloud visualization to visually represent each cluster's dominant themes.

API Rate Limiting

Spotify and Genius APIs have rate limits that can interrupt user experience.

Solution

Implemented batch API calls, caching strategies, and graceful error handling to maintain smooth UX.
